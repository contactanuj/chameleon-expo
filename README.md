# The Chameleon - pass-and-play (Expo + WebView)

An unofficial, fan-made companion app for the social bluffing game **The Chameleon**
(Big Potato / Crown & Andrews). One device is passed around the table; the app is the
**only** thing you need - no cards, no dice. It privately shows every player the secret
word/picture except the Chameleon, then runs the clues, debate, vote, the catch, the
final guess and the scoring.

Built with the same pattern as the sibling apps (`wink-killer-expo`, `secret-hitler-expo`):
a **pure rules engine** + **DOM UI** + **CSS**, inlined into a single `app.html` that an
Expo `react-native-webview` shell loads. Everything in one file → trivial to ship as an APK.

## Both editions, in one APK
- **Word edition** - the classic Topic Cards: a 4×4 grid of 16 words (Food, Animals, Movies, ...).
- **Picture edition** - the same game with a 4×4 grid of **bundled OpenMoji illustrations** (Animals, Faces, Sea Life, ...).
- **Mixed** - draw from both.
- **Custom topics** - make your own 16-word or emoji grids on-device (the dry-wipe "custom card").

> Picture-edition artwork is **OpenMoji** (CC BY-SA 4.0), inlined as SVG by
> `scripts/fetch-openmoji.js` into `assets/ch-art.js` - so the single-file build is
> preserved (no loose binary assets). Each grid cell is a string; cells with no bundled
> art (e.g. custom emoji topics) fall back to the system emoji glyph.

## Everything is configurable
Sensible defaults match the rulebook; the **Advanced** panel exposes the rest, with live
validation (errors block an unplayable game; warnings flag off-spec-but-fine choices):

- Players (2-12) & names (seating order = clue order)
- Number of Chameleons (1 = official; >1 = team variant, warned)
- Clue phase on/off · optionally record each spoken clue for a recap
- Separate debate step · **hide the grid during debate** (the 7-8 player variant)
- Voting mode: **table** (point IRL, record it) · **open** (tap each vote) · **secret** (pass to vote)
- Tie-breaker: dealer decides · re-vote (capped, can't hang) · Chameleon escapes
- Chameleon's guesses when caught (+ auto **2 guesses at exactly 3 players**, per the rulebook)
- **Bots** - switch any seat to a computer player (fill seats / solo practice), with easy/medium/hard difficulty
- Optional **timers** - a per-player clue timer (the box's sand timer) and a debate countdown (non-binding)
- Scoring on/off, win target, and each point value (escape / caught-but-guessed / caught-failed)
- Dealer rotation: the caught Chameleon deals next (rulebook) · clockwise · random
- Cosmetic A1-D4 grid coordinates

## How a round plays
1. **Reveal** - pass the device; each player privately sees the secret word, or learns they're 🦎.
2. **Clues** - clockwise from the dealer, each says one related word.
3. **Debate** - argue it out (optionally with the grid face-down).
4. **Vote** - accuse a suspect.
5. **Reveal** - wrong target ⇒ Chameleon escaped; right target ⇒ the Chameleon gets a final **guess**.
6. **Score** - escape: Chameleon +2 · caught-but-guessed: Chameleon +1 · caught-and-wrong: everyone else +2. First to 5 wins.

## Project layout
```
assets/
  styles.css      inlined <style>
  ch-engine.js    pure rules engine (no DOM/network; unit-tested)
  ch-content.js   built-in topic library + bot clue knowledge (BOT_CLUES)
  ch-art.js       GENERATED: inline OpenMoji illustrations (CC BY-SA 4.0)
  ch-bots.js      pure offline bot decisions (clue/bluff/vote/guess; unit-tested)
  ui.js           pass-and-play DOM UI
  app.html        GENERATED by build.js (do not edit by hand)
  icon.png        app/adaptive icon - clean chameleon silhouette (1024×1024)
tests/
  engine.test.js  config validation, scoring, tie-breakers, fuzzed full matches
  bots.test.js    clue-knowledge invariant, suspicion model, competence, bot/mixed fuzz
  ui.test.js      headless drive of every screen, voting mode, info-leak guards, solo-vs-bots
scripts/
  make-icon.ps1       regenerates assets/icon.png (pure GDI+, no external tools)
  fetch-openmoji.js   regenerates assets/ch-art.js from OpenMoji (needs network)
App.js, app.json, eas.json, babel.config.js, build.js
```

## Develop / test / build
```bash
npm install
npm test            # engine + headless UI tests
npm run build:html  # compose assets/app.html from the source modules
npm start           # build:html then expo start
npm run build:android   # build:html then EAS preview APK
```
`npm start`/`android`/`build:android` rebuild `app.html` first, so editing
`ch-engine.js` / `ch-content.js` / `ui.js` / `styles.css` is enough - never hand-edit `app.html`.

### Add topics
Edit `assets/ch-content.js` (`w(id,name,category,[16 items])` for words,
`p(...)` for emoji) and rebuild - the test asserts every built-in topic has exactly 16 items.
